# 问题修复记录

## 修复1：游戏时间流速过快

**日期：** 2026-06-25  
**文件：** `data/components/info.py`  
**行号：** 第300行

### 问题描述
游戏右上角倒计时时间流逝速度约为正常速度的2.5倍，每秒实际减少约2.5秒。

### 原因分析
`update_count_down_clock()` 方法中使用 `pg.time.get_ticks()` 获取毫秒级时间，但阈值设为 `400`：
```python
elif (level_info[c.CURRENT_TIME] - self.current_time) > 400:
```
400毫秒 = 0.4秒减1，导致计时器约2.5倍速运行。

### 修复内容
将阈值从 `400` 改为 `1000`（1秒）：
```python
# 修改前：
elif (level_info[c.CURRENT_TIME] - self.current_time) > 400:

# 修改后：
elif (level_info[c.CURRENT_TIME] - self.current_time) > 1000:
```

## 修复2：马里奥跳跃高度不足

**日期：** 2026-06-25  
**文件：** `data/constants.py`  
**行号：** 第42行

### 问题描述
马里奥跳跃高度不足（约135像素），无法跳过游戏中最大的管道障碍物（pipe3/pipe4，顶部距地面172像素）。

### 原因分析
初始跳跃速度 `JUMP_VEL = -10` 配合 `JUMP_GRAVITY = 0.31`，在60FPS下计算得最大跳跃高度约135像素，而大三管道顶部距地面172像素，差距约37像素。

### 修复内容
将跳跃初速度从 -10 提高到 -12：
```python
# 修改前：
JUMP_VEL = -10

# 修改后：
JUMP_VEL = -12
```
修改后最大跳跃高度约170像素，可跳过所有管道障碍物。

## 修复3：主菜单背景音乐

**日期：** 2026-06-25  
**文件：** `data/states/main_menu.py`  
**行号：** 第33行、第132行

### 问题描述
主菜单界面没有背景音乐，游戏启动后进入主菜单是静音状态。

### 原因分析
原代码中 `Sound` 对象仅在 `Level1` 启动时创建，播放 `main_theme`，而主菜单状态（`Menu`）没有任何音乐管理。

### 修复内容
在主菜单中直接管理音乐生命周期：

**`startup()` 末尾（第33行）：**
```python
# 新增：
pg.mixer.music.load(setup.MUSIC['title'])
pg.mixer.music.play(-1)  # -1 = 无限循环
```

**`reset_game_info()` 末尾（第132行）：**
```python
# 新增：
pg.mixer.music.stop()
```

进入主菜单时循环播放 title.ogg，玩家按开始键时立即停止，随后进入关卡时 `Sound` 自动播放 main_theme。
